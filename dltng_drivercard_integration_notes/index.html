    Integration in das bestehende DLTNG Download Tool¶

    Neue Module¶

    • ddd_drivercard_parser.py
    • ui_drivercard_analysis.py

    Zweck¶

    • Erweiterung der vorhandenen SQLite-Archivdatenbank um Analyse-Tabellen für Fahrerkarte-Imports
    • Nachträgliches Einlesen von .DDD-Dateien aus ddd_archive
    • Aggregation von Lenkzeit, Ruhezeit, Bereitschaft und sonstiger Arbeit nach Tag, Woche oder Doppelwoche
    • Separates PySide6-Fenster für Auswertung

    Minimaler Einbau in ui_mainwindow.py¶

    1. Import ergänzen¶

    from ui_drivercard_analysis import DriverCardAnalysisDialog

    2. Menü oder Button ergänzen¶

    Zum Beispiel in _build_menu(): 

    a_drivercard_analysis = QAction("Fahrerkarten-Auswertung", self)
a_drivercard_analysis.triggered.connect(self.open_drivercard_analysis)
datei_menu.addSeparator()
datei_menu.addAction(a_drivercard_analysis)

    3. Methode in MainWindow ergänzen¶

    def open_drivercard_analysis(self):
    sqlite_db = Path(self.config_manager.get_app_settings().sqlite_db)
    dlg = DriverCardAnalysisDialog(sqlite_db, self)
    dlg.exec()

    Wichtiger Hinweis zum Parser¶

    Der native Python-Fallback in ddd_drivercard_parser.py importiert nur sicher erkennbare Metadaten. Für normgerechte Aktivitätsintervalle ist ein externer Vollparser vorgesehen.

    Dazu vor dem Start des Programms eine Umgebungsvariable setzen, z. B.: 

    set DLTNG_TACHO_CMD=tachograph parse --input {input} --output {output}

    Der externe Befehl muss: - {input} und {output} unterstützen - JSON als Ausgabedatei schreiben - Aktivitäten als Intervalle liefern
