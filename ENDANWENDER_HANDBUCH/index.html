    DLTNG Download Tool - Endanwender-Handbuch¶

    Zweck des Handbuchs¶

    Dieses Handbuch beschreibt die Bedienung des DLTNG Download Tools über die grafische Benutzeroberfläche. Es richtet sich an Endanwender, Support-Mitarbeiter und Administratoren, die Geräte verwalten, Downloads einrichten, DDD-Dateien archivieren und Fahrerkarten auswerten.

    Grundbegriffe¶

    • Quelle: Ein System, von dem Dateien abgerufen werden. Das kann ein FTP/FTPS-Server oder eine DLT API sein.
    • Task: Eine geplante oder manuell gestartete Download-Aufgabe. Ein Task nutzt eine oder mehrere Quellen.
    • DDD-Datei: Tachographen- oder Fahrerkartendatei im digitalen Tachographenformat.
    • Archiv: Komprimierte und protokollierte Ablage importierter DDD-Dateien.
    • Audit-Log: Nachvollziehbares Protokoll wichtiger Aktionen.
    • Fahrerkarte: DDD-Datei mit fahrerbezogenen Aktivitäten, Ereignissen, Störungen, Fahrzeug- und Positionsdaten.

    Programmstart¶

    1. Starten Sie die Anwendung über die bereitgestellte Programmdatei oder aus der Entwicklungsumgebung mit python main.py.
    2. Nach dem Start öffnet sich das Hauptfenster DLTNG Download Tool.
    3. In der Statusleiste unten wird der aktuelle Zustand angezeigt, z. B. Bereit.
    4. Die wichtigsten Funktionen befinden sich in den Reitern der Hauptansicht.

    Aufbau der Oberfläche¶

    Die Hauptoberfläche ist in folgende Reiter gegliedert:

    • Geräteverwaltung
    • Quellen
    • Tasks
    • DDD-Archiv
    • Audit-Log
    • Einstellungen
    • Logs
    • Rohkonfiguration

    Zusätzliche Funktionen finden Sie im Menü, insbesondere:

    • Fahrerkarten-Auswertung
    • Ungeparste Archivdaten nachverarbeiten
    • Archiv-Datenbank bereinigen
    • DriverCard REST API starten
    • DriverCard REST API stoppen
    • DriverCard REST API im Browser öffnen
    • DriverCard REST API Log anzeigen
    • DDD Wochenansicht Web-Demo

    Erste Einrichtung¶

    Einstellungen prüfen¶

    1. Öffnen Sie den Reiter Einstellungen. 2. Prüfen Sie das Logverzeichnis. 3. Prüfen Sie das Log-Level. Für den normalen Betrieb ist INFO sinnvoll. 4. Prüfen Sie die Anzahl der aufzubewahrenden Logdateien. 5. Prüfen Sie die SQLite-Datei. Dort werden Archiv-, Audit-, Geräte- und Fahrerkartendaten gespeichert. 6. Prüfen Sie das DDD-Archiv-Verzeichnis. 7. Tragen Sie den Benutzernamen für Audit ein. Dieser Name erscheint später im Audit-Log. 8. Aktivieren oder deaktivieren Sie den automatischen Start der DriverCard REST API. 9. Klicken Sie auf Einstellungen speichern.

    Geräteverwaltung¶

    Der Reiter Geräteverwaltung verwaltet Download-Terminals und deren Status.

    Übersicht¶

    Im oberen Bereich sehen Sie Kennzahlen, z. B. Geräte gesamt und aktive Geräte. Darunter befinden sich Filter und die Gerätetabelle.

    Die Gerätetabelle enthält unter anderem:

    • ID
    • Name
    • Kunde
    • Standort
    • Status
    • Aktiv
    • Letzter Erfolg
    • Gruppe(n)
    • URL
    • Info
    • Dateien

    Wenn Sie ein Gerät auswählen, erscheinen darunter Detailinformationen in mehreren Unterreitern:

    • Stammdaten
    • Statushistorie
    • Audit
    • Gruppen

    Geräte suchen und filtern¶

    1. Geben Sie im Feld Suche einen Gerätenamen, eine URL, Seriennummer, Inventarnummer oder ein Tag ein.
    2. Wählen Sie optional Kunde, Standort, Gruppe, Status oder Aktiv.
    3. Klicken Sie auf Filter anwenden.
    4. Mit Aktualisieren laden Sie die Übersicht neu.

    Mandanten verwalten¶

    1. Klicken Sie auf Mandanten verwalten. 2. Im Dialog können Sie Mandanten mit Neu, Bearbeiten und Löschen pflegen. 3. Speichern Sie Ihre Änderung.

    Nur Benutzer mit Administratorrechten können Mandanten verwalten.

    Standorte verwalten¶

    1. Klicken Sie auf Standorte verwalten. 2. Legen Sie neue Standorte an oder bearbeiten Sie bestehende Einträge. 3. Ordnen Sie Standorte bei Bedarf einem Mandanten zu.

    Nur Benutzer mit Administratorrechten können Standorte verwalten.

    Gruppen verwalten¶

    1. Klicken Sie auf Gruppen verwalten. 2. Legen Sie Gruppen an, z. B. nach Region, Kunde oder Gerätetyp. 3. Gruppen können später Geräten zugeordnet werden.

    Nur Benutzer mit Administratorrechten können Gruppen verwalten.

    Gerät anlegen¶

    1. Klicken Sie auf Gerät anlegen. 2. Tragen Sie den Gerätename ein. 3. Wählen Sie optional Kunde/Mandant, Standort und Gruppen. 4. Ergänzen Sie technische Angaben wie Gerätetyp, Seriennummer, URL, Status-Endpunkt und weitere Stammdaten. 5. Speichern Sie den Dialog. 6. Das Gerät erscheint anschließend in der Gerätetabelle.

    Gerät bearbeiten¶

    1. Wählen Sie ein Gerät in der Tabelle aus.
    2. Klicken Sie auf Gerät bearbeiten.
    3. Ändern Sie Stammdaten, Zuordnung oder Verbindungseinstellungen.
    4. Speichern Sie den Dialog.
    5. Die Änderung wird im Geräte-Audit protokolliert.

    Gerät duplizieren¶

    1. Wählen Sie ein vorhandenes Gerät aus.
    2. Klicken Sie auf Gerät duplizieren.
    3. Das Tool legt eine Kopie des Geräts an.
    4. Bearbeiten Sie anschließend den neuen Datensatz, z. B. Name, Seriennummer oder URL.

    Gerät deaktivieren¶

    1. Wählen Sie ein Gerät aus.
    2. Klicken Sie auf Gerät deaktivieren.
    3. Bestätigen Sie die Sicherheitsabfrage.
    4. Das Gerät bleibt sichtbar, wird aber als inaktiv geführt.

    Gerät archivieren¶

    1. Wählen Sie ein Gerät aus.
    2. Klicken Sie auf Gerät archivieren.
    3. Bestätigen Sie die Sicherheitsabfrage.
    4. Das Gerät wird archiviert und deaktiviert.

    Statuscheck für ein Gerät¶

    1. Wählen Sie ein Gerät aus.
    2. Klicken Sie auf Statuscheck.
    3. Das Tool ruft den konfigurierten Status-Endpunkt des Geräts ab.
    4. Nach Abschluss erscheint ein Dialog mit Status und Antwortdetails.
    5. Die Statushistorie wird aktualisiert.

    Mögliche Statuswerte sind z. B. ONLINE, BUSY, OFFLINE, API_UNREACHABLE, AUTH_FAILED, WARNING, ERROR, MAINTENANCE und DISABLED.

    Statuscheck für Treffer¶

    1. Setzen Sie zunächst die gewünschten Filter.
    2. Klicken Sie auf Statuscheck für Treffer.
    3. Das Tool prüft nacheinander alle aktuell gefilterten Geräte.
    4. Der Fortschrittsdialog zeigt pro Gerät Status und Detailmeldung.

    System-Informationen anzeigen¶

    1. Klicken Sie in der Gerätetabelle in der Spalte Info auf die Schaltfläche. 2. Das Tool liest die Systeminformationen vom Gerät aus. 3. Im Dialog System-Informationen sehen Sie die Informationen als Tabelle. 4. Wählen Sie eine Zeile und klicken Sie auf Wert kopieren, um den Wert in die Zwischenablage zu kopieren.

    Datei-Explorer eines Geräts¶

    1. Klicken Sie in der Gerätetabelle in der Spalte Dateien auf die Schaltfläche. 2. Der Dialog Datei-Explorer lädt die Dateiliste des Geräts. 3. Mit Aktualisieren laden Sie die Liste neu. 4. Wählen Sie eine Datei und klicken Sie auf Herunterladen, um sie lokal zu speichern. 5. Wählen Sie eine Datei und klicken Sie auf Löschen, um sie auf dem Gerät zu löschen. 6. Bestätigen Sie Löschabfragen nur, wenn die Datei wirklich entfernt werden soll.

    Website-Logfile auslesen¶

    1. Wählen Sie ein Gerät aus.
    2. Klicken Sie auf Website-Logfile auslesen.
    3. Wählen Sie, falls angeboten, das gewünschte Logfile.
    4. Das Logfile wird in einem Dialog angezeigt.

    Quellen¶

    Im Reiter Quellen verwalten Sie die Systeme, von denen Dateien geladen werden.

    Die Tabelle zeigt:

    • ID
    • Typ
    • Name
    • Host
    • Port
    • Protokoll
    • Verzeichnis
    • Löschen

    Quelle anlegen¶

    1. Öffnen Sie den Reiter Quellen. 2. Klicken Sie auf Quelle anlegen. 3. Wählen Sie den Quellentyp, z. B. FTP oder DLT_API. 4. Tragen Sie einen Namen ein. 5. Tragen Sie Host, Port und Protokoll ein. 6. Tragen Sie Benutzername und Passwort oder einen API-Key ein. 7. Tragen Sie das Remote-Verzeichnis ein. 8. Aktivieren Sie TLS, wenn die Verbindung verschlüsselt erfolgen soll. 9. Aktivieren oder deaktivieren Sie Passiver Modus. 10. Legen Sie den Timeout fest. 11. Aktivieren Sie Datei nach Download löschen nur, wenn Quelldateien nach erfolgreichem Abruf entfernt werden sollen. 12. Klicken Sie optional auf Testverbindung. 13. Klicken Sie auf Speichern.

    Quelle bearbeiten¶

    1. Wählen Sie eine Quelle in der Tabelle aus. 2. Klicken Sie auf Quelle bearbeiten. 3. Ändern Sie die gewünschten Felder. 4. Testen Sie die Verbindung bei Bedarf. 5. Speichern Sie die Änderung.

    Quelle löschen¶

    1. Wählen Sie eine Quelle aus.
    2. Klicken Sie auf Quelle löschen.
    3. Bestätigen Sie die Abfrage.

    Hinweis: Prüfen Sie vor dem Löschen, ob Tasks diese Quelle noch verwenden.

    Verbindung testen¶

    1. Wählen Sie eine Quelle aus.
    2. Klicken Sie auf Testverbindung.
    3. Das Tool prüft die Erreichbarkeit und Zugangsdaten.
    4. Das Ergebnis wird als Meldung angezeigt.

    Tasks¶

    Im Reiter Tasks verwalten Sie geplante und manuelle Download-Aufgaben.

    Die Tabelle zeigt:

    • ID
    • Name
    • Aktiv
    • Typ
    • Zeit
    • Quellen
    • Filter
    • DDD-Archiv
    • Letzter Status
    • Letzte Meldung

    Task anlegen¶

    1. Öffnen Sie den Reiter Tasks. 2. Klicken Sie auf Task anlegen. 3. Geben Sie einen Namen ein. 4. Aktivieren Sie den Task, wenn er automatisch ausgeführt werden soll. 5. Wählen Sie den Planungstyp: - hourly: stündliche Ausführung zur angegebenen Minute. - daily: tägliche Ausführung zur angegebenen Stunde und Minute. - weekly: wöchentliche Ausführung am angegebenen Wochentag zur angegebenen Stunde und Minute. 6. Setzen Sie Minute, Stunde und bei wöchentlicher Planung Wochentag. 7. Wählen Sie eine oder mehrere Quellen aus. 8. Tragen Sie Dateifilter ein, z. B. *.DDD;*.ddd;*.csv. 9. Wählen Sie das lokale Downloadverzeichnis. 10. Aktivieren Sie Rekursiv, wenn Unterverzeichnisse einbezogen werden sollen. 11. Aktivieren Sie DDD revisionssicher archivieren, wenn DDD-Dateien automatisch archiviert werden sollen. 12. Wählen Sie die Dublettenprüfung: - hash: Prüfung anhand des SHA-256-Dateihashes. - hash_and_name: Prüfung anhand Hash und Dateiname. 13. Klicken Sie auf Speichern.

    Task bearbeiten¶

    1. Wählen Sie einen Task aus.
    2. Klicken Sie auf Task bearbeiten.
    3. Ändern Sie Planung, Quellen, Filter, Zielverzeichnis oder Archivoptionen.
    4. Speichern Sie den Dialog.

    Task löschen¶

    1. Wählen Sie einen Task aus.
    2. Klicken Sie auf Task löschen.
    3. Bestätigen Sie die Sicherheitsabfrage.

    Task sofort starten¶

    1. Wählen Sie einen Task aus.
    2. Klicken Sie auf Task sofort starten.
    3. Ein Fortschrittsdialog zeigt die Quellen, Statuswerte und Ergebnisse.
    4. Nach Abschluss werden Letzter Status und Letzte Meldung in der Taskliste aktualisiert.
    5. Wenn Archivierung aktiviert ist, werden DDD-Dateien automatisch ins Archiv übernommen.

    DDD-Archiv¶

    Der Reiter DDD-Archiv zeigt archivierte DDD-Dateien und deren Metadaten.

    Archiv durchsuchen¶

    1. Öffnen Sie den Reiter DDD-Archiv.
    2. Wählen Sie im Bereich Suche / Filter einen Zeitraum.
    3. Tragen Sie optional einen Teil des Dateinamens ein.
    4. Aktivieren Sie optional die Anzeige gelöschter Datensätze.
    5. Klicken Sie auf Suchen.
    6. Mit Filter zurücksetzen kehren Sie zur Standardansicht zurück.

    Die Tabelle zeigt:

    • Archiv-ID
    • Archivdatum
    • Dateiname
    • Originalgröße
    • Komprimiert
    • Quelle
    • Task
    • Status
    • Archivpfad

    Archiv aktualisieren¶

    Klicken Sie auf Aktualisieren, um die Tabelle neu aus der Datenbank zu laden.

    Archivierte Datei exportieren¶

    1. Wählen Sie einen Archivdatensatz aus.
    2. Klicken Sie auf Ausgewählte Datei exportieren.
    3. Wählen Sie Zielordner und Dateiname.
    4. Das Tool entpackt die archivierte .ddd.gz-Datei und schreibt eine DDD-Datei.
    5. Der Export wird im Audit-Log dokumentiert.

    Archivdatensatz löschen¶

    1. Wählen Sie einen Archivdatensatz aus.
    2. Klicken Sie auf Ausgewählten Datensatz löschen.
    3. Geben Sie einen Löschgrund ein.
    4. Bestätigen Sie den Vorgang.
    5. Der Datensatz wird logisch gelöscht und im Audit-Log protokolliert.

    Hinweis: Ein logisch gelöschter Eintrag ist in der Datenbank weiterhin nachvollziehbar.

    Details und Audit anzeigen¶

    1. Wählen Sie einen Archivdatensatz aus.
    2. Klicken Sie auf Details / Audit.
    3. Der Dialog zeigt Dateiname, Archiv-ID, Archivdatum, Größen, Quelle, Task, Remote-Pfad, Archivpfad, Löschstatus und Audit-Historie.

    Fahrerkarten-Auswertung öffnen¶

    1. Wählen Sie optional einen Archivdatensatz aus. 2. Klicken Sie auf Fahrerkarten-Auswertung öffnen. 3. Die Fahrerkarten-Auswertung öffnet sich. 4. Wenn möglich, wird der gewählte Archivdatensatz direkt vorausgewählt.

    Ungeparste Archivdaten nachverarbeiten¶

    1. Klicken Sie auf Ungeparste Archivdaten jetzt nachverarbeiten.
    2. Das Tool sucht archivierte DDD-Dateien, die noch nicht in die Fahrerkartenanalyse importiert wurden.
    3. Bestätigen Sie den Vorgang.
    4. Nach Abschluss werden Archiv- und Analyseansichten aktualisiert.

    Audit-Log¶

    Der Reiter Audit-Log zeigt protokollierte Aktionen aus der Datenbank.

    Audit-Log filtern¶

    1. Öffnen Sie Audit-Log.
    2. Wählen Sie Von und Bis.
    3. Tragen Sie optional einen Benutzer ein.
    4. Tragen Sie optional einen Action-Type ein.
    5. Nutzen Sie Textsuche für Archiv-ID, Datei, Karte oder Details.
    6. Klicken Sie auf Audit-Log aktualisieren.
    7. Mit Filter zurücksetzen laden Sie wieder die Standardansicht.

    Die Tabelle zeigt:

    • Zeitpunkt
    • Benutzer
    • Action-Type
    • Archiv-ID
    • Karten-Nr.
    • Datei
    • Nachname
    • Vorname
    • Führerschein-Nr.
    • Details

    Logs¶

    Der Reiter Logs zeigt technische Logdateien.

    Logdatei anzeigen¶

    1. Öffnen Sie Logs.
    2. Klicken Sie auf Logs aktualisieren.
    3. Wählen Sie links eine Logdatei aus.
    4. Rechts wird der Inhalt angezeigt.

    DriverCard REST API Log anzeigen¶

    1. Öffnen Sie über das Menü DriverCard REST API Log anzeigen.
    2. Das passende Logfile wird im Log-Reiter ausgewählt, sofern es vorhanden ist.

    Fahrerkarten-Auswertung¶

    Die Fahrerkarten-Auswertung wird entweder über das Menü oder aus dem Reiter DDD-Archiv geöffnet.

    Aufbau der Auswertung¶

    Oben sehen Sie die Liste Importierte Fahrerkarten. Darunter befinden sich Filter und Ergebnis-Reiter.

    Die Importliste zeigt unter anderem:

    • Karten- und Fahrerinformationen
    • Zeitraum der Aktivitäten
    • Archiv-ID
    • Datei
    • Importstatus
    • Parserinformationen

    Die Ergebnisansicht enthält:

    • Zeitenauswertung
    • Ereignisse
    • Störungen
    • Benutzte Fahrzeuge
    • Positionsdaten

    Zusätzliche DDD-Datei einlesen¶

    1. Öffnen Sie die Fahrerkarten-Auswertung.
    2. Klicken Sie auf Zusätzliche DDD-Datei einlesen.
    3. Wählen Sie eine oder mehrere DDD-Dateien aus.
    4. Das Tool archiviert und importiert die Dateien.
    5. Nach Abschluss zeigt ein Dialog verarbeitete Dateien, importierte Dateien und Dubletten.

    Importliste aktualisieren¶

    Klicken Sie auf Liste aktualisieren, um die Liste der importierten Fahrerkarten neu zu laden.

    Auswahl neu einlesen¶

    1. Wählen Sie eine oder mehrere importierte Fahrerkarten in der Importliste.
    2. Klicken Sie auf Auswahl neu einlesen.
    3. Bestätigen Sie den Hinweis.
    4. Bestehende Analyse-, Ereignis- und Störungsdaten dieser Archive werden ersetzt.

    Alle neu einlesen¶

    1. Klicken Sie auf Alle neu einlesen.
    2. Bestätigen Sie den Hinweis.
    3. Das Tool liest alle importierten Fahrerkarten erneut ein.

    Importierte Fahrerkarte dauerhaft löschen¶

    1. Wählen Sie eine oder mehrere importierte Fahrerkarten aus.
    2. Klicken Sie auf Auswahl dauerhaft löschen.
    3. Bestätigen Sie die Sicherheitsabfrage.
    4. Das Tool löscht Import-, Aktivitäts-, Ereignis-, Störungs-, Fahrzeug- und Positionsdaten.
    5. Der Vorgang wird im Audit-Log protokolliert.

    Kontextmenü der Importliste¶

    In der Importliste können Sie mit der rechten Maustaste ein Kontextmenü öffnen.

    Verfügbare Aktionen:

    • Aktivitätsdaten anzeigen
    • Tagesübersicht anzeigen
    • Wochenübersicht anzeigen
    • Ausgewählte Fahrerkarte dauerhaft löschen

    Ein Doppelklick auf eine importierte Fahrerkarte übernimmt die Filter und startet die Auswertung.

    Filter setzen¶

    1. Wählen Sie Von und Bis.
    2. Wählen oder erfassen Sie optional:
    3. Karten-Nr.
    4. Nachname
    5. Vorname
    6. Führerschein-Nr.
    7. Wählen Sie die Gruppierung, z. B. Tag, Woche oder Monat.
    8. Klicken Sie auf Auswertung aktualisieren.

    Zeitenauswertung anzeigen¶

    1. Setzen Sie die gewünschten Filter.
    2. Klicken Sie auf Auswertung aktualisieren.
    3. Öffnen Sie den Reiter Zeitenauswertung.
    4. Die obere Tabelle zeigt aggregierte Zeiten.
    5. Die Detailtabelle zeigt einzelne Aktivitätsdatensätze.

    Typische Aktivitätstypen:

    • DRIVING: Lenkzeit
    • WORK: Arbeit
    • AVAILABILITY: Bereitschaft
    • BREAK_REST: Fahrtunterbrechung oder Ruhe
    • REST: Ruhezeit

    Aktivitätsdaten anzeigen¶

    1. Setzen Sie die gewünschten Filter.
    2. Klicken Sie auf Aktivitätsdaten anzeigen.
    3. Ein separates Fenster zeigt die einzelnen Aktivitätsdaten chronologisch.

    Tagesübersicht anzeigen¶

    1. Setzen Sie den gewünschten Zeitraum und Fahrerfilter. 2. Klicken Sie auf Tagesübersicht anzeigen. 3. Das Tool öffnet eine Tages-Timeline der Fahreraktivitäten. 4. Prüfen Sie Lenk-, Arbeits-, Bereitschafts- und Ruhezeiten im Tagesverlauf.

    Wenn für den gewählten Zeitraum keine Aktivitäten vorhanden sind, zeigt das Tool eine entsprechende Meldung.

    Wochenübersicht anzeigen¶

    1. Setzen Sie den gewünschten Zeitraum und Fahrerfilter. 2. Klicken Sie auf Wochenübersicht anzeigen. 3. Das Tool öffnet eine Wochen-Timeline. 4. Nutzen Sie diese Ansicht, um Aktivitäten über mehrere Tage hinweg zu prüfen.

    Ereignisse auswerten¶

    1. Setzen Sie die gewünschten Filter.
    2. Klicken Sie auf Auswertung aktualisieren.
    3. Öffnen Sie den Reiter Ereignisse.
    4. Die Tabelle zeigt Ereigniszeitpunkt, Fahrer, Gruppe, Typ, Code, Kennzeichen und Beschreibung.

    Störungen auswerten¶

    1. Setzen Sie die gewünschten Filter.
    2. Klicken Sie auf Auswertung aktualisieren.
    3. Öffnen Sie den Reiter Störungen.
    4. Die Tabelle zeigt Störungszeitpunkt, Fahrer, Gruppe, Typ, Code, Kennzeichen und Beschreibung.

    Benutzte Fahrzeuge anzeigen¶

    1. Setzen Sie die gewünschten Filter.
    2. Klicken Sie auf Auswertung aktualisieren.
    3. Öffnen Sie den Reiter Benutzte Fahrzeuge.
    4. Die Tabelle zeigt Fahrzeuge, Nutzungszeiträume und fahrerbezogene Zuordnung.

    Positionsdaten anzeigen¶

    1. Setzen Sie die gewünschten Filter.
    2. Klicken Sie auf Auswertung aktualisieren.
    3. Öffnen Sie den Reiter Positionsdaten.
    4. Die Tabelle zeigt Positionszeitpunkte, Quelle, Typ, Kartennummer, Fahrer, Land, Region, Breiten- und Längengrad sowie Details.

    Karte öffnen¶

    1. Öffnen Sie den Reiter Positionsdaten. 2. Klicken Sie auf Karte öffnen. 3. Wenn Positionsdaten mit Koordinaten vorhanden sind, öffnet das Tool die Kartenansicht. 4. Wenn keine Koordinaten vorhanden sind, erscheint eine Meldung.

    EU 561/2006 Verstöße prüfen¶

    1. Setzen Sie Zeitraum und Fahrerfilter. 2. Klicken Sie auf EU 561/2006 Verstöße. 3. Das Tool öffnet die technische Vorprüfung nach VO (EG) Nr. 561/2006. 4. Prüfen Sie die Zusammenfassung. 5. Wählen Sie einen Verstoß aus, um Details und Rechtsgrundlage anzuzeigen. 6. Prüfen Sie zusätzlich die Hinweise.

    Die Prüfung berücksichtigt typische Fälle wie:

    • Fahrtunterbrechung nach Lenkzeitblöcken
    • tägliche Lenkzeit
    • verlängerte tägliche Lenkzeit
    • wöchentliche Lenkzeit
    • Doppelwochen-Lenkzeit
    • tägliche Ruhezeit
    • wöchentliche Ruhezeit

    Hinweis: Die Auswertung ist eine technische Unterstützung. Die finale rechtliche Bewertung muss fachlich geprüft werden.

    Verstöße exportieren¶

    1. Öffnen Sie die Verstoßauswertung.
    2. Klicken Sie auf Verstöße exportieren.
    3. Wählen Sie Dateiname und Format.
    4. Je nach Auswahl wird der Bericht als JSON, CSV oder Textdatei gespeichert.

    Zeitauswertung exportieren¶

    1. Öffnen Sie die Fahrerkarten-Auswertung.
    2. Setzen Sie die gewünschten Filter.
    3. Klicken Sie im Reiter Zeitenauswertung auf Zeitauswertung exportieren.
    4. Wählen Sie ein Dateiformat.
    5. Speichern Sie die Datei.

    Aktivitätsdatensätze exportieren¶

    1. Öffnen Sie die Fahrerkarten-Auswertung.
    2. Setzen Sie die gewünschten Filter.
    3. Klicken Sie auf Aktivitätsdatensätze exportieren.
    4. Wählen Sie Speicherort und Format.

    DriverCard REST API¶

    Die lokale DriverCard REST API dient zur technischen Bereitstellung von Parser- und Auswertungsfunktionen.

    REST API starten¶

    1. Öffnen Sie das Menü.
    2. Klicken Sie auf DriverCard REST API starten.
    3. Das Tool zeigt Basis-URL, Health-Endpunkt und Parse-Endpunkt an.

    Typische Endpunkte:

    • /health
    • /api/drivercard/parse

    REST API stoppen¶

    1. Öffnen Sie das Menü.
    2. Klicken Sie auf DriverCard REST API stoppen.
    3. Das Tool beendet die lokale API.

    REST API im Browser öffnen¶

    1. Öffnen Sie das Menü.
    2. Klicken Sie auf DriverCard REST API im Browser öffnen.
    3. Falls die API noch nicht läuft, bietet das Tool den Start an.
    4. Danach wird der Health-Endpunkt im Browser geöffnet.

    Wochenansicht Web-Demo öffnen¶

    1. Öffnen Sie das Menü.
    2. Klicken Sie auf DDD Wochenansicht Web-Demo.
    3. Falls nötig, startet das Tool die DriverCard REST API.
    4. Die Web-Demo für die Wochenansicht wird geöffnet.

    Archiv-Datenbank bereinigen¶

    Die Archivbereinigung entfernt Archivdatensätze, deren physische Archivdatei fehlt, und schreibt ein Bereinigungslog.

    1. Öffnen Sie das Menü.
    2. Klicken Sie auf Archiv-Datenbank bereinigen.
    3. Lesen Sie die Sicherheitsabfrage sorgfältig.
    4. Starten Sie die Bereinigung nur, wenn fehlende physische Archivdateien wirklich aus der Datenbank entfernt werden sollen.
    5. Nach Abschluss zeigt das Tool eine Zusammenfassung, z. B. gelöschte Archiv-, Import-, Aktivitäts-, Ereignis-, Störungs-, Fahrzeug- und Positionsdatensätze.
    6. Der Vorgang wird im Audit-Log protokolliert.

    Typische Arbeitsabläufe¶

    Neuen Download einrichten¶

    1. Öffnen Sie Quellen.
    2. Legen Sie eine Quelle an und testen Sie die Verbindung.
    3. Öffnen Sie Tasks.
    4. Legen Sie einen Task an.
    5. Wählen Sie Quelle, Dateifilter, Zielverzeichnis und Planung.
    6. Aktivieren Sie bei DDD-Dateien die revisionsorientierte Archivierung.
    7. Speichern Sie den Task.
    8. Starten Sie den Task einmal manuell mit Task sofort starten.
    9. Prüfen Sie das Ergebnis in DDD-Archiv, Audit-Log und Logs.

    Fahrerkarte manuell auswerten¶

    1. Öffnen Sie DDD-Archiv.
    2. Klicken Sie auf Fahrerkarten-Auswertung öffnen.
    3. Klicken Sie auf Zusätzliche DDD-Datei einlesen.
    4. Wählen Sie die DDD-Datei.
    5. Warten Sie den Import ab.
    6. Wählen Sie die Fahrerkarte in der Importliste.
    7. Setzen Sie Zeitraum und Filter.
    8. Klicken Sie auf Auswertung aktualisieren.
    9. Nutzen Sie Tagesansicht, Wochenansicht oder Verstoßauswertung.

    Download-Ergebnis prüfen¶

    1. Öffnen Sie Tasks.
    2. Prüfen Sie Letzter Status und Letzte Meldung.
    3. Öffnen Sie DDD-Archiv, um archivierte Dateien zu prüfen.
    4. Öffnen Sie Audit-Log, um Archiv-, Export- oder Löschaktionen nachzuvollziehen.
    5. Öffnen Sie Logs, wenn technische Details oder Fehler benötigt werden.

    Gerät prüfen und Datei vom Gerät laden¶

    1. Öffnen Sie Geräteverwaltung.
    2. Suchen oder filtern Sie das gewünschte Gerät.
    3. Klicken Sie auf Statuscheck.
    4. Prüfen Sie die Statushistorie.
    5. Öffnen Sie bei Bedarf System-Informationen.
    6. Öffnen Sie den Datei-Explorer.
    7. Wählen Sie die gewünschte Datei und klicken Sie auf Herunterladen.

    Fehlerbehebung¶

    Quelle ist nicht erreichbar¶

    • Prüfen Sie Host, Port und Protokoll.
    • Prüfen Sie Benutzername, Passwort oder API-Key.
    • Prüfen Sie, ob TLS und passiver FTP-Modus zur Gegenstelle passen.
    • Nutzen Sie Testverbindung.
    • Prüfen Sie die Logdateien.

    Task lädt keine Dateien¶

    • Prüfen Sie, ob der Task aktiv ist.
    • Prüfen Sie die zugeordneten Quellen.
    • Prüfen Sie Dateifilter und Remote-Verzeichnis.
    • Prüfen Sie, ob die Dateien bereits verarbeitet wurden.
    • Starten Sie den Task manuell und lesen Sie die Fortschrittsmeldungen.

    DDD-Datei erscheint nicht in der Fahrerkarten-Auswertung¶

    • Prüfen Sie, ob die Datei im DDD-Archiv vorhanden ist.
    • Klicken Sie auf Ungeparste Archivdaten jetzt nachverarbeiten.
    • Öffnen Sie die Fahrerkarten-Auswertung und klicken Sie auf Liste aktualisieren.
    • Prüfen Sie die Logs auf Parserfehler.

    Keine Positionsdaten auf der Karte¶

    • Prüfen Sie im Reiter Positionsdaten, ob Datensätze vorhanden sind.
    • Die Karte kann nur geöffnet werden, wenn Breiten- und Längengrad vorhanden sind.
    • Nicht jede Fahrerkarte enthält Positionsdaten mit Koordinaten.

    Audit-Log ist leer¶

    • Prüfen Sie den Zeitraumfilter.
    • Klicken Sie auf Filter zurücksetzen.
    • Prüfen Sie, ob die konfigurierte SQLite-Datei korrekt ist.

    Hinweise zum sicheren Arbeiten¶

    • Löschen Sie Archiv- oder Fahrerkartendatensätze nur, wenn der Vorgang fachlich gewollt ist.
    • Nutzen Sie Löschgründe aussagekräftig, da sie im Audit-Log erscheinen.
    • Prüfen Sie vor dem Löschen von Gerätedateien im Datei-Explorer, ob diese noch benötigt werden.
    • Bearbeiten Sie die Rohkonfiguration nur mit ausreichender Kenntnis der JSON-Struktur.
    • Verwenden Sie die Verstoßauswertung als technische Vorprüfung und nicht als alleinige rechtliche Entscheidung.
