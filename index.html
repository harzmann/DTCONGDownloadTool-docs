    FLEET Mira documentation¶

    FLEET Mira brings device operations, compliant tachograph archiving, driver-card and vehicle-unit analysis, fleet records, driving-licence checks, deadlines, finance documents and verified cloud backup into one Windows application.

    Download – version 1.1.0-beta5¶

    Current prerelease: FLEET Mira 1.1.0-beta5, released on 22 July 2026 for Windows x64.

    Download setup Download portable EXE Download portable ZIP

    The setup package is recommended for a system-wide installation and stores shared application data below %PROGRAMDATA%\eh-systemhaus\FLEET Mira. The portable edition stores its runtime data beside the executable. Create a current backup of existing application data before upgrading.

    Key features¶

    FLEET Mira combines compliant tachograph-data processes, fleet management, cost control and traceable compliance workflows in one central application. Its key capabilities include:

    • Tachograph-data archiving and analysis
      • automated recursive import of C/M files from USB removable drives (Pro 💎)
      • day and week charts for personnel and vehicles
      • interactive maps for position data
      • technical infringement evaluation under Regulation (EC) No 561/2006
      • cryptographic signature verification for driver cards and vehicle units
      • audit trail for EU and German FPersV procedures
      • support for current Smart Tachograph 2 data:
        • border crossings
        • loading and unloading operations
        • load type
        • authenticated GNSS positions
        • Generation 2 Version 2 files
        • 56-day data structures of new driver cards
    • Fleet Management
      • RFID-assisted driving-licence checks
      • personnel due-date management, including driver cards, professional-driver qualification and ADR
      • vehicle due-date management, including tachograph calibration, statutory inspections and service
      • cost controlling, KPIs and TCO (Total Cost of Ownership)
        • fixed operating costs such as insurance and tax
        • variable operating costs such as maintenance and repairs
      • XRechnung and e-invoice import with audit-ready original-document storage
      • reporting:
        • cost per vehicle
        • cost per kilometre
        • fixed and variable costs
        • time and period comparisons
      • audit trail for GoBD-compliant procedures
    • Users and roles with granular permissions and optional two-factor authentication (2FA)
    • Multilingual and regional formatting
      • Languages: Deutsch, English, Français, Italiano, Español, Português, Polski, Română, Nederlands, Dansk, Čeština, Magyar, Slovenščina, Hrvatski, Suomi, Norsk bokmål, Svenska, Latviešu, Lietuvių, Eesti, Slovenčina, Български and Ελληνικά
      • Regions: Germany (de-DE), Austria (de-AT), Switzerland (de-CH), United Kingdom (en-GB), United States (en-US), France (fr-FR), Italy (it-IT), Spain (es-ES), Portugal (pt-PT), Poland (pl-PL), Romania (ro-RO), Netherlands (nl-NL), Denmark (da-DK), Czechia (cs-CZ), Hungary (hu-HU), Slovenia (sl-SI), Croatia (hr-HR), Finland (fi-FI), Norway (nb-NO), Sweden (sv-SE), Latvia (lv-LV), Lithuania (lt-LT), Estonia (et-EE), Slovakia (sk-SK), Bulgaria (bg-BG) and Greece (el-GR)
    • Optional backup of local tachograph and financial archives plus the database to filesystem, WebDAV or Google Drive targets, including restore workflows
    • Application-wide auditing with one central, filterable audit log
    • Complete compliance history for imports, checks, assignments, changes and exports
    • GDPR-compliant data processing through role-based access, protected credentials and traceable procedures

    Key principles¶

    • Original C and M files remain byte-for-byte traceable through hashes and the immutable local archive.
    • Signature results are stored per signed section; invalid or unverifiable files are quarantined without being deleted.
    • Business changes and automatic imports are recorded in the global audit log.
    • All tachograph calculations use UTC. Region settings change formatting only.
    • Optional backup is one-way from the local primary archives to one configured target.
    • Locally verified financial originals can be verified and booked independently of backup status.
    • UI layout, language and region are stored per user.

    Product and legacy names¶

    FLEET Mira is the visible product name. Technical identifiers such as DLTNGDownloadTool.spec, database tables, API paths, existing configuration keys and the backup prefix DLTNG/<installation-id>/... remain unchanged for compatibility. They must not be renamed in an existing installation.
