    DLTNG Download Tool - RESTful API¶

    Ueberblick¶

    Das DLTNG Download Tool stellt eine lokale RESTful API fuer Fahrerkarten-DDD-Dateien und bereits importierte Fahrerkartendaten bereit. Die API wird aus der Desktop-Anwendung gestartet und verwendet die in den Einstellungen konfigurierte SQLite-Datenbank.

    Standard-Basis-URL:

    http://127.0.0.1:5055

    Alle Endpunkte liefern JSON. Erfolgreiche Antworten enthalten ok: true; Fehlerantworten enthalten ok: false und error.

    Start in der Anwendung¶

    Die API wird ueber das Menue bedient:

    • DriverCard REST API starten
    • DriverCard REST API stoppen
    • DriverCard REST API im Browser oeffnen
    • DriverCard REST API Log anzeigen
    • DDD Wochenansicht Web-Demo

    Wenn REST API beim Start automatisch starten in den Einstellungen aktiv ist, startet die API beim Programmstart automatisch.

    Gemeinsame Filter¶

    Die datenbankbasierten Auswertungsendpunkte unterstuetzen diese Query-Parameter:

    Parameter Beschreibung Beispiel
    date_from Startdatum im Format YYYY-MM-DD; Standard ist heute minus 30 Tage. 2026-04-01
    date_to Enddatum im Format YYYY-MM-DD; Standard ist heute. 2026-04-30
    last_name Nachname. Ohne % wird automatisch eine Enthalten-Suche genutzt. Davies
    first_name Vorname. Ohne % wird automatisch eine Enthalten-Suche genutzt. John
    card_number Kartennummer. Ohne % wird automatisch eine Enthalten-Suche genutzt. UTO22
    driving_license_number Fuehrerscheinnummer. Ohne % wird automatisch eine Enthalten-Suche genutzt. ABC123

    Eigene SQL-LIKE-Muster koennen mit % oder _ uebergeben werden, z. B. last_name=%Dav%.

    Antwortformate¶

    Listenendpunkte antworten typischerweise so:

    {
  "ok": true,
  "data": {
    "type": "activities",
    "count": 12,
    "filters": {
      "date_from": "2026-04-01",
      "date_to": "2026-04-30",
      "last_name": "",
      "first_name": "",
      "card_number": "",
      "driving_license_number": ""
    },
    "items": []
  }
}

    Fehlerantwort:

    {
  "ok": false,
  "error": "Beschreibung des Fehlers"
}

    Health Check¶

    GET /health¶

    Prueft, ob die API laeuft.

    curl http://127.0.0.1:5055/health

    Antwort:

    {
  "ok": true,
  "service": "drivercard_rest_api",
  "host": "127.0.0.1",
  "port": 5055,
  "timestamp": "2026-04-29T10:15:00"
}

    Fahrerkarte direkt parsen¶

    POST /api/drivercard/parse¶

    Parst eine Fahrerkarte direkt aus einem Upload oder aus Base64-Inhalt. Dieser Endpunkt benoetigt keine vorherige Archivierung.

    Multipart Upload:

    curl -X POST http://127.0.0.1:5055/api/drivercard/parse -F "file=@C:\temp\fahrerkarte.DDD"

    Base64 JSON:

    {
  "file_name": "fahrerkarte.DDD",
  "ddd_base64": "BASE64_INHALT"
}

    Die Antwort enthaelt card, activities, week_index, events, faults, vehicles_used, positions und Parser-Metadaten.

    Importierte Fahrerkarten¶

    GET /api/drivercard/imports¶

    Liefert importierte Fahrerkarten inklusive Archivbezug und Aktivitaetszeitraum.

    curl http://127.0.0.1:5055/api/drivercard/imports

    Wichtige Felder: archive_id, imported_at, file_name, card_number, holder_name, first_name, last_name, driving_license_number, activity_count, activity_from, activity_to.

    Zeitauswertung¶

    GET /api/drivercard/time-summary¶

    Liefert aggregierte Aktivitaetszeiten.

    Zusaetzlicher Parameter:

    Parameter Beschreibung Werte
    grouping Gruppierung der Auswertung; Standard ist day. day, week, month 
    curl "http://127.0.0.1:5055/api/drivercard/time-summary?date_from=2026-04-01&date_to=2026-04-30&grouping=week"

    Wichtige Felder: period, card_number, last_name, first_name, holder_name, driving_minutes, work_minutes, availability_minutes, rest_minutes, total_minutes, interval_count.

    Aktivitaetsdatensaetze¶

    GET /api/drivercard/activities¶

    Liefert einzelne Aktivitaetsintervalle.

    curl "http://127.0.0.1:5055/api/drivercard/activities?date_from=2026-04-01&date_to=2026-04-30&last_name=Davies"

    Wichtige Felder: id, archive_id, card_number, last_name, first_name, activity_date, start_ts, end_ts, activity_type, duration_minutes, crew, slot.

    Aktivitaetstypen:

    • DRIVING: Lenkzeit
    • WORK: Arbeit
    • AVAILABILITY: Bereitschaft
    • BREAK_REST: Pause/Ruhe
    • REST: Ruhe

    Ereignisse¶

    GET /api/drivercard/events¶

    Liefert Ereignisse aus importierten Fahrerkarten.

    curl "http://127.0.0.1:5055/api/drivercard/events?date_from=2026-04-01&date_to=2026-04-30"

    Wichtige Felder: event_ts, card_number, last_name, first_name, event_group, event_type_name, event_type_display, event_code, vehicle_registration, place_text.

    Stoerungen¶

    GET /api/drivercard/faults¶

    Liefert Stoerungen aus importierten Fahrerkarten.

    curl "http://127.0.0.1:5055/api/drivercard/faults?date_from=2026-04-01&date_to=2026-04-30"

    Wichtige Felder: fault_ts, card_number, last_name, first_name, fault_group, fault_type_name, fault_type_display, fault_code, vehicle_registration, place_text.

    Benutzte Fahrzeuge¶

    GET /api/drivercard/vehicles-used¶

    Liefert Fahrzeuge, die in den Fahrerkartendaten verwendet wurden.

    curl "http://127.0.0.1:5055/api/drivercard/vehicles-used?date_from=2026-04-01&date_to=2026-04-30"

    Wichtige Felder: first_use_ts, last_use_ts, card_number, last_name, first_name, vehicle_registration, vin, odometer_begin, odometer_end, distance_km, vu_data_block_counter.

    Positionsdaten¶

    GET /api/drivercard/positions¶

    Liefert Positionsdaten aus EF.Places und EF.GNSS_Places, sofern vorhanden.

    curl "http://127.0.0.1:5055/api/drivercard/positions?date_from=2026-04-01&date_to=2026-04-30"

    Wichtige Felder: record_ts, position_ts, card_number, last_name, first_name, source_type, entry_type, country_text, region_text, odometer_value, latitude, longitude, gnss_accuracy, authentication_status, map_label.

    Tagesuebersicht¶

    GET /api/drivercard/day-overview¶

    Liefert Aktivitaeten nach Kalendertagen gruppiert. Dieser Endpunkt ist fuer Tages-Timelines geeignet.

    curl "http://127.0.0.1:5055/api/drivercard/day-overview?date_from=2026-04-24&date_to=2026-04-24&card_number=UTO22"

    Jeder Eintrag enthaelt period, date, weekday, totals und activities.

    Beispiel fuer totals:

    {
  "driving_minutes": 120,
  "work_minutes": 60,
  "availability_minutes": 0,
  "rest_minutes": 300,
  "total_minutes": 480,
  "interval_count": 8
}

    Wochenuebersicht¶

    GET /api/drivercard/week-overview¶

    Liefert Aktivitaeten nach ISO-Kalenderwochen gruppiert. Innerhalb jeder Woche werden die Aktivitaeten zusaetzlich nach Tagen gruppiert.

    curl "http://127.0.0.1:5055/api/drivercard/week-overview?date_from=2026-04-20&date_to=2026-04-26"

    Jeder Eintrag enthaelt period, date_from, date_to, totals, days und activities.

    EU 561/2006 Verstoesse¶

    GET /api/drivercard/eu561/violations¶

    Fuehrt eine technische Vorpruefung nach VO (EG) Nr. 561/2006 aus.

    Zusaetzlicher Parameter:

    Parameter Beschreibung Werte
    multi_manning Bewertet bestimmte Ruhezeitfenster fuer Mehrfahrerbetrieb; Standard ist false. true, false, 1, 0 
    curl "http://127.0.0.1:5055/api/drivercard/eu561/violations?date_from=2026-04-01&date_to=2026-04-30&card_number=UTO22"

    Die Antwort enthaelt card_number, holder_name, analyzed_from, analyzed_to, total_intervals, count, warnings, stats, legal_notice und violations.

    Wichtige Felder je Verstoss:

    • rule_code
    • severity
    • article
    • title
    • description
    • start_ts
    • end_ts
    • actual_minutes
    • allowed_minutes
    • delta_minutes
    • context

    Hinweis: Die EU-561-Auswertung ist eine technische Vorpruefung. Sonderfaelle, Ausnahmen und die rechtliche Wuerdigung muessen fachlich geprueft werden.

    HTTP Statuscodes¶

    Status Bedeutung
    200 Anfrage erfolgreich.
    400 Ungueltige Parameter, z. B. falsches Datum oder unbekannte Gruppierung.
    500 Interner Fehler, z. B. Datenbank- oder Parserfehler.
